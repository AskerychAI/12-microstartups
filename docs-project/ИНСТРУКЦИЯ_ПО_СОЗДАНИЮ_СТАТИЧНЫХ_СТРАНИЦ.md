# Инструкция по созданию и добавлению статичных страниц в меню

## Обзор

Эта инструкция описывает процесс создания статичных страниц (например, "Концепция проекта", "Правила участия" и т.д.) и их добавления в главное меню сайта.

## Структура файлов

Все статичные страницы размещаются в папке:
```
content/pages/
```

## Шаг 1: Создание файла страницы

1. Создайте новый файл Markdown в папке `content/pages/`
2. Имя файла должно быть на латинице (рекомендуется использовать транслитерацию или английский)
   - ✅ Хорошо: `concept.md`, `rules.md`, `pravila-uchastiya.md`
   - ❌ Плохо: `концепция.md`, `правила.md`

## Шаг 2: Заполнение front matter

Каждая страница должна начинаться с front matter (метаданных) в формате YAML:

```yaml
---
title: "Название страницы (отображается на странице)"
date: 2025-01-27T00:00:00+03:00
draft: false
description: "Краткое описание страницы для SEO"
---
```

### Параметры front matter:

- **title** - Заголовок страницы (обязательно)
- **date** - Дата создания (обязательно, формат: YYYY-MM-DDTHH:MM:SS+03:00)
- **draft** - Черновик (false = опубликовано, true = скрыто)
- **description** - Описание для поисковых систем (опционально)

## Шаг 3: Написание содержимого

После front matter добавьте содержимое страницы в формате Markdown:

```markdown
# Заголовок первого уровня

Текст страницы...

## Подзаголовок

Еще текст...
```

## Шаг 4: Добавление страницы в меню

Чтобы страница появилась в главном меню сайта, нужно добавить её в файл `hugo.toml`.

### Откройте файл `hugo.toml` в корне проекта

Найдите секцию `[[menu.main]]` и добавьте новую запись:

```toml
[[menu.main]]
  name = "Название в меню"
  pageRef = "/pages/имя-файла"
  weight = 20
```

### Параметры меню:

- **name** - Текст, который будет отображаться в меню
- **pageRef** - Путь к странице (без расширения .md)
  - Формат: `/pages/имя-файла`
  - Пример: если файл называется `concept.md`, то `pageRef = "/pages/concept"`
- **weight** - Порядок отображения в меню (чем меньше число, тем выше в меню)

## Примеры

### Пример 1: Страница "Концепция проекта"

**Файл:** `content/pages/concept.md`

```markdown
---
title: "Концепция проекта"
date: 2025-01-27T00:00:00+03:00
draft: false
description: "Описание концепции проекта 12 microstartups"
---

# Концепция проекта

Здесь описание концепции...

## Цели проекта

Текст о целях...
```

**Добавление в `hugo.toml`:**

```toml
[[menu.main]]
  name = "Концепция"
  pageRef = "/pages/concept"
  weight = 1
```

### Пример 2: Страница "Правила участия"

**Файл:** `content/pages/rules.md`

```markdown
---
title: "Правила участия"
date: 2025-01-27T00:00:00+03:00
draft: false
description: "Правила участия в проекте 12 microstartups"
---

# Правила участия

Здесь правила...
```

**Добавление в `hugo.toml`:**

```toml
[[menu.main]]
  name = "Правила участия"
  pageRef = "/pages/rules"
  weight = 2
```

## Порядок действий (чеклист)

- [ ] Создать файл `.md` в папке `content/pages/`
- [ ] Заполнить front matter (title, date, draft, description)
- [ ] Написать содержимое страницы
- [ ] Открыть `hugo.toml`
- [ ] Добавить запись `[[menu.main]]` с правильными параметрами
- [ ] Указать уникальный `weight` для правильного порядка
- [ ] Проверить, что `pageRef` соответствует имени файла (без .md)

## Важные замечания

1. **Имена файлов**: Используйте латиницу для имен файлов, чтобы избежать проблем с URL
2. **Weight**: Убедитесь, что каждый пункт меню имеет уникальный `weight`
3. **pageRef**: Путь должен начинаться с `/pages/` и не содержать расширение `.md`
4. **Проверка**: После добавления страницы запустите локальный сервер Hugo для проверки

## Текущая структура меню

Текущие пункты меню (для справки):

- `weight = 10` - "Политика обработки данных" (`/privacy`)

Рекомендуемые значения weight для новых страниц:
- `weight = 1-9` - Основные страницы (Концепция, Правила и т.д.)
- `weight = 10+` - Вспомогательные страницы (Политика и т.д.)

## Проверка результата

После создания страницы и добавления в меню:

1. Запустите локальный сервер: `hugo server`
2. Откройте сайт в браузере
3. Проверьте, что страница появилась в меню
4. Перейдите на страницу и убедитесь, что содержимое отображается корректно

## Решение проблем

### Страница не отображается в меню
- Проверьте синтаксис в `hugo.toml` (правильные скобки `[[menu.main]]`)
- Убедитесь, что `pageRef` указан правильно
- Проверьте, что файл существует в `content/pages/`

### Страница не открывается
- Проверьте, что `pageRef` соответствует имени файла (без .md)
- Убедитесь, что файл имеет правильный front matter
- Проверьте, что `draft: false` в front matter

### Неправильный порядок в меню
- Проверьте значения `weight` - меньшее число = выше в меню
- Убедитесь, что каждый пункт имеет уникальный `weight`
